## Release Artifacts

The release workflow currently produces these platform bundles:

| Platform | Typical files |
| --- | --- |
| Linux | `.deb`, `.AppImage`, `.rpm` |
| macOS Apple Silicon | `aarch64.dmg` |
| macOS Intel | `x64.dmg` |
| Windows | NSIS `.exe`, sometimes `.msi` |
| All platforms | `novywave-chrome` binary |

## Chrome Desktop Launcher

The Chrome Desktop Launcher (`novywave-chrome`) is a lightweight alternative to the standard desktop application. It starts the NovyWave backend and opens the interface in Chrome, Chromium, or Microsoft Edge using `--app` mode.

This option is useful when:
- WebKitGTK rendering is slow on your Linux system
- You prefer Chrome's rendering engine
- You want a minimal installation without platform-specific GUI dependencies

Download `novywave-chrome` for your platform from the [GitHub Releases](https://github.com/NovyWave/NovyWave/releases) page.

## Automatic Updates

NovyWave includes a built-in update mechanism for packaged desktop releases. When a newer release is available, the application can prompt you to update without manually searching for a download.

## Verifying Your Installation

After installation, launch NovyWave. You should see the main window with three panels:

1. **Files & Scopes** (top-left) — Load and browse waveform files
2. **Variables** (top-right) — Select signals to display
3. **Selected Variables** (bottom) — View waveform timeline

Try loading a waveform file using the **Load Files** button to verify everything works.
